There is a long journey to become a DevOps engineer. So let's start it!

Below are the pillars that need to be strong enough to sustain a successful career as a DevOps engineer (actually the term DevOps engineer is still debated... but we will go with it anyhow).

So what does the term mean anyhow? According to Wikipedia "DevOps is a set of practices that combines software development (Dev) and IT operations (Ops). It aims to shorten the systems development life cycle and provide continuous delivery with high software quality". Fancy words, in reality it means there are is a ton to know from both Dev as well as from Ops. And on top on that, there is a strong need to have some people skills too, as collaboration with a lot of other professionals is needed.

Programming

Any DevOps professional must learn to code because automation is achieved through coding. Some programming languages do stand out here.

Operating Systems

As part of the DevOps roadmap, you need to learn the OS concepts such as Windows, UNIX, and Linux. Both Linux and Git have the same author.

Network and Security

Being aware of the networking concepts and managing devices such as Switches, Firewall, IDS/IPS, and routers, etc. is also required.

Servers

Automation of the server management is key to DevOps as these servers provide the necessary platform for the applications.

Infrastructure-as-a-code

There are various dev, test, and prod environments that consist of a lot of infrastructures that need to be configured, automated, managed, monitored, and logged.

CI/CD

Applications are built, tested, and deployed in production by automating the workflow.

Monitoring

It is crucial to know how your application is performing, especially in production environments. For this we need to monitor it with various tools.

Cloud

Many of the companies are transforming from on-premises to cloud and using cloud services so that they spend only the services offered and not on the infrastructure.
